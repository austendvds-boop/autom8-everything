# B1: Environment & Health Foundation

## Context
autom8everything.com is deployed on Vercel with Neon Postgres. The `NEXT_PUBLIC_SITE_URL` env var is NOT set in production, causing magic link URLs, Stripe redirect URLs, and SMS funnel links to default to `http://localhost:3000`. The Cadence onboarding checkout route calls a non-existent endpoint. There's no unified health endpoint.

## Tasks

### 1. Add `/api/health` endpoint
Create `src/app/api/health/route.ts` that checks:
- DB connectivity (run `SELECT 1` via both `rfDb` and `platformDb`)
- All critical env vars are present (not empty):
  - `NEXT_PUBLIC_SITE_URL`
  - `DATABASE_URL`
  - `STRIPE_SECRET_KEY`
  - `A8_ADMIN_SECRET`, `A8_JWT_SECRET`
  - `RF_ENCRYPTION_KEY`, `RF_JWT_SECRET`
  - `CRON_SECRET`
  - `CADENCE_API_URL`, `PORTAL_API_SECRET`
- Table existence checks: run `SELECT to_regclass('public.a8_clients')` and `SELECT to_regclass('public.rf_consent_log')` — report null if missing
- Cadence v2 reachability: `fetch(CADENCE_API_URL)` with 5s timeout, check for `{"status":"ok"}`
- Return JSON: `{ healthy: boolean, checks: { ... }, timestamp: string }`
- Use `export const dynamic = "force-dynamic"` and `export const runtime = "nodejs"`

### 2. Fix Cadence onboarding checkout route
The file `src/app/api/onboarding/checkout/route.ts` currently calls `${cadenceBaseUrl}/api/stripe/checkout` which doesn't exist on cadence-v2.

Replace the entire route implementation:
- Instead of proxying to cadence-v2's non-existent Stripe checkout, use the existing `createPortalCheckoutSession` from `@/lib/platform/services/stripe-portal`
- Accept the same fields the current `CadenceGetStartedClient.tsx` sends: `business_name`, `owner_name`, `owner_email`, `owner_phone`, `hours`, `business_description`, `faqs`, `area_code`, `transfer_number`
- Map to portal checkout: `product: "cadence"`, `email: owner_email`, `businessName: business_name`, `phone: owner_phone`, `areaCode: area_code`
- Return `{ url: session.url }`
- Keep CORS headers and error handling

### 3. Add NEXT_PUBLIC_SITE_URL safety check to configs
In `src/lib/platform/config.ts` and `src/lib/review-funnel/config.ts`:
- Change the `NEXT_PUBLIC_SITE_URL` default from `"http://localhost:3000"` to a production-safe default:
  - `.default("https://autom8everything.com")`
- This ensures if the env var somehow isn't set, URLs still work (even though we're also setting it in Vercel)

### 4. Update vercel.json with NEXT_PUBLIC branding vars as defaults
This is NOT needed — env vars are set in Vercel dashboard, not in vercel.json. Skip this.
