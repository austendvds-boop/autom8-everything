# B4: SMS Graceful Degradation & Launch Checklist

## Context
The Twilio A2P campaign was rejected, meaning automated SMS sending will fail at the Twilio API level. The Review Funnel's core value is sending post-appointment review requests — if SMS is dead, there needs to be a fallback. The `process-sms` cron already handles failures gracefully (marks as failed after 3 attempts), but customers need visibility into why their SMS aren't sending.

## Tasks

### 1. Add email-based review request fallback
Create `src/lib/review-funnel/services/email-review-request.ts`:
- Export `sendReviewRequestEmail(reviewRequestId: string): Promise<void>`
- Fetch the review request, tenant, and location
- Build a branded HTML email (reuse the style from `magic-link-email.ts` shell)
- Subject: `{business_name} — We'd love your feedback`
- Body: link to `{NEXT_PUBLIC_SITE_URL}/r/{reviewRequestId}` with a "Leave a Review" button
- Use the same `resolveSmtpCredentials()` + `nodemailer` pattern
- Send from `aust@autom8everything.com`

### 2. Update process-sms cron to use email fallback
In `src/app/api/review-funnel/cron/process-sms/route.ts`:
- After calling `sendReviewRequest()`, if the result is `"sent"`, done (SMS worked)
- If SMS fails with a Twilio error (catch block), AND the review request has a `customerEmail`:
  - Try `sendReviewRequestEmail(row.reviewRequestId)` 
  - If email succeeds, update pending_sms status to `"sent_email"` (add this as a new valid status)
  - If email also fails, mark as failed per existing logic
- If no customerEmail, mark as failed per existing logic
- Add `sentEmail` counter to the response JSON

### 3. Add SMS status indicator to dashboard
In `src/app/review-funnel/dashboard/DashboardOverviewClient.tsx` (or the main dashboard component):
- Add a banner/alert at the top when SMS is not working:
  - Check a new API endpoint `/api/review-funnel/dashboard/sms-status` 
  - If the last 5 SMS attempts all failed → show yellow banner: "SMS delivery is temporarily limited. Review requests will be sent via email when possible."
- Keep it informational, not alarming

### 4. Create SMS status API
Create or update `src/app/api/review-funnel/dashboard/sms-usage/route.ts` (it may already exist):
- Add a `smsHealthy` boolean to the response
- Query: count of `rf_pending_sms` where `status = 'failed'` AND `created_at > now() - interval '24 hours'`
- If >5 failures and 0 successes in last 24h → `smsHealthy: false`
